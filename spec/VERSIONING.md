# HJS 协议版本编号机制

本协议使用 **语义化版本 2.0.0**（SemVer）进行版本管理。

版本格式：`主版本号.次版本号.修订号`

- **主版本号**：当做了不兼容的 API 或数据结构变更时递增。
- **次版本号**：当以向下兼容的方式添加功能时递增。
- **修订号**：当进行向下兼容的问题修正时递增。

**协议版本与实现的兼容性承诺**：
- 在同一个主版本号下，所有符合规范的实现必须能够互操作。
- 协议规范本身也遵循 SemVer，即主版本号变化表示协议不向后兼容。

**当前版本**：v1.0.0（初始发布）

协议版本的变更必须通过 [RFC 流程](rfcs/) 进行提议和记录。
