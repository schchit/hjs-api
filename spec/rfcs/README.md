# HJS 协议 RFC 流程

本目录用于记录协议规范的变更提议。

## 流程步骤

1. **创建 RFC 草案**
   - 复制 `0000-template.md` 为 `rfcs/xxxx-my-feature.md`（其中 xxxx 为下一个可用编号）。
   - 填写草案内容，状态设为“草案”。
   - 提交 Pull Request 到本仓库。

2. **讨论阶段**
   - 在 PR 中或 GitHub Issues 中收集反馈。
   - 作者根据讨论更新 RFC。

3. **采纳/拒绝**
   - 维护者（目前为项目创始人）决定是否采纳。
   - 如果采纳，状态改为“已采纳”，并分配版本号。
   - 如果拒绝，状态改为“已拒绝”并说明理由。

4. **实现**
   - 采纳的 RFC 应在后续协议版本中实现。
   - 实现完成后，更新 RFC 为“已完成”。

5. **存档**
   - 所有 RFC 无论采纳与否都永久保留，以便追溯历史。
